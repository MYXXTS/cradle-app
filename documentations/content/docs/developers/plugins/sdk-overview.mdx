# Plugin SDK Overview

这一页概述 Cradle Plugin SDK 的稳定边界。插件必须通过 manifest 显式声明能力与权限，运行时不会为缺失的 `contributes` 字段补默认值。

## Manifest Contract

`cradle.contributes` 是必填对象，用来描述插件暴露的 capability 和需要的 permission。即使插件暂时不声明 capability 或 permission，也要写空数组。

```yaml
key: cradle.contributes
type: "{ capabilities: CapabilityContribution[]; permissions: PermissionContribution[] }"
required: true
```

## Minimal Package

```json
{
  "name": "@example/cradle-plugin",
  "version": "0.1.0",
  "cradle": {
    "contributes": {
      "capabilities": [],
      "permissions": []
    }
  }
}
```

能力声明和权限声明共同构成 host activation 的审计输入。外部本地插件在激活前必须满足 permission grant；插件代码不能依赖未声明的运行时 capability。
